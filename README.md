# ИЗ-1: «Кладезь мудрости» — Homework 01

Проект содержит две самостоятельные версии программы, которые читают файл с афоризмами и пословицами, выполняют проверку данных и формируют результирующий файл. Первая реализация выполнена в процедурном стиле, вторая использует классы и фабрику.

## Структура каталога
```text
Homework01/
├── procedural/       # процедурная реализация
├── oop/              # объектно-ориентированная реализация
├── tests/            # тестовые входные файлы и эталонные выходы
└── README.md
```

## Сборка и запуск
Команды следует выполнять из каталога `Homework01/`.

```bash
# процедурная версия
c++ -std=c++17 procedural/main.cpp procedural/wisdom.cpp -o procedural/procedural_app
./procedural/procedural_app tests/input3.txt tests/output_proc.txt

# объектно-ориентированная версия
c++ -std=c++17 oop/*.cpp -o oop/oop_app
./oop/oop_app tests/input3.txt tests/output_oop.txt
```

## Формат входного файла
Каждая строка имеет вид
```
<type> "<origin>" "<content>"
```
`type` равен `aphorism` или `proverb`. Поле `origin` содержит автора либо страну, `content` — текст высказывания. Строки с неизвестным типом или нарушенной структурой пропускаются с сообщением об ошибке.

## Аргументы командной строки
Программа принимает два аргумента: путь к входному файлу и путь к файлу с полным выводом. Третий параметр зарезервирован под последующие задания.

## Тестовые данные
| Файл | Назначение |
| --- | --- |
| `input1.txt` | Пустой файл, проверяется реакция на отсутствие данных. |
| `input2.txt` | Один корректный афоризм, минимальный рабочий пример. |
| `input3.txt` | Набор из трех афоризмов и двух пословиц. |
| `input4.txt` | Незнакомый тип `legend`, ожидается диагностическое сообщение. |
| `input5.txt` | Строка с ошибкой в кавычках, тест на устойчивость парсера. |
| `input6.txt` | Заготовка для следующих версий, применима после расширения набора типов. |

Для каждого сценария подготовлен эталонный вывод, что облегчает проверку корректности работы программы.
