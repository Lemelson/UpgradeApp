# ИЗ-1: «Кладезь мудрости» — Homework 02

Во второй версии к афоризмам и пословицам добавлена альтернатива `riddle` (загадка). Формат строки неизменен: появился дополняющий тип, в котором поле `origin` хранит ответ. Процедурная реализация расширена за счет дополнительного `case`, объектно-ориентированная включает новый класс `Riddle` и его регистрацию в фабрике.

## Структура каталога
```text
Homework02/
├── procedural/
├── oop/
├── tests/
└── README.md
```

## Сборка и запуск
Команды выполняются в каталоге `Homework02/`.

```bash
# процедурная версия
c++ -std=c++17 procedural/main.cpp procedural/wisdom.cpp -o procedural/procedural_app
./procedural/procedural_app tests/input3.txt tests/output_proc.txt

# объектно-ориентированная версия
c++ -std=c++17 oop/*.cpp -o oop/oop_app
./oop/oop_app tests/input3.txt tests/output_oop.txt

# пример с загадкой
./procedural/procedural_app tests/input6.txt tests/output_proc_riddle.txt
./oop/oop_app tests/input6.txt tests/output_oop_riddle.txt
```

## Формат входного файла
```
<type> "<origin>" "<content>"
```
Типы `aphorism` и `proverb` интерпретируются как прежде. Тип `riddle` использует поле `origin` для хранения ответа.

## Аргументы командной строки
Программа по-прежнему принимает два аргумента: путь к входному файлу и путь к полному выводу.

## Метрики
| Версия | Файлов | Примерно строк кода |
| --- | --- | --- |
| Процедурная | 3 | 250 |
| Объектно-ориентированная | 15 | 512 |

## Тестовые данные
| Файл | Назначение |
| --- | --- |
| `input1.txt` | Пустой файл, проверка реакции на отсутствие данных. |
| `input2.txt` | Один афоризм, минимальный рабочий пример. |
| `input3.txt` | Базовый набор без загадок, регрессионная проверка. |
| `input4.txt` | Незнакомый тип `legend`, ожидается диагностическое сообщение. |
| `input5.txt` | Ошибка в кавычках, тест на устойчивость парсера. |
| `input6.txt` | Набор с загадкой, афоризмом и пословицей, демонстрирующий новую альтернативу. |

Для каждого сценария подготовлены эталонные выходы в каталоге `tests/`.
